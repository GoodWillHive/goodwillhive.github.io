Cut Your AI Token Bill by 60%
Without Losing Quality

Giảm 60% Chi Phí Token AI
Mà Không Mất Chất Lượng

The average person wastes 40–70% of their AI token budget. Not because they ask hard questions — because they ask them badly.

1. Cut the preamble

Most prompts start with politeness. "Hi! I hope you're doing well. I was wondering if you could possibly help me with..." Stop. The AI doesn't care. Every word costs money.

Before: "Hi Claude! I hope you're doing well today. I was wondering if you could help me write a brief summary of the following article..."

After: "Summarize in 3 bullets:"

Same result. ~60% fewer tokens in the prompt.

2. Specify output length

Without constraints, AI fills space. "Write a report" produces 800 words when you needed 200. Always add: Max 200 words. or 3 bullets only.

Constraining output length is the single highest-ROI prompt change you can make. One word saves hundreds.

3. Use structured input, not prose

Prose prompts force the AI to parse meaning. Structured prompts give it directly:

Prose: "I need you to review this email I wrote to a client where I'm asking for payment that's overdue by 30 days but I don't want to sound aggressive..."

Structured:

  • Task: review email
  • Context: 30-day overdue payment
  • Tone: firm but not aggressive
  • Output: revised email only, no explanation

The structured version uses fewer tokens AND produces a better result.

4. Don't repeat context

In a long session, people re-explain the same thing in every message. "As I mentioned, this is for a Vietnamese restaurant..." — if you said it 3 messages ago, it's still in context. Stop saying it again.

5. Use caching for repeated system prompts

If you use the API: Claude, GPT-4, and Gemini all support prompt caching. A 2,000-token system prompt cached costs ~10% of the uncached price per call. Set it once, reference forever.

Quick wins summary:
Remove greetings → −20 tokens avg
Add length constraint → −40% output tokens avg
Use structured input → −25% prompt tokens avg
Stop repeating context → −30% in long sessions
Enable caching → −90% on system prompt per call

The mindset shift

You are not writing an email to a human. You are writing code that gets compiled. Every word either adds meaning or adds cost. Cut everything that adds only cost.

Người dùng AI trung bình lãng phí 40–70% ngân sách token. Không phải vì câu hỏi khó — mà vì cách đặt câu hỏi quá tệ.

1. Cắt phần mở đầu không cần thiết

Hầu hết prompt bắt đầu bằng lịch sự. "Chào Claude! Mình muốn nhờ bạn giúp một chút nếu được..." Dừng lại. AI không quan tâm đến điều này. Mỗi từ đều tốn tiền.

Trước: "Chào! Mình đang cần tóm tắt bài báo sau đây nếu bạn có thể giúp được..."

Sau: "Tóm tắt 3 điểm chính:"

Kết quả như nhau. Ít hơn ~60% token trong prompt.

2. Giới hạn độ dài output

Không có ràng buộc, AI sẽ viết dài. "Viết báo cáo" ra 800 từ khi bạn chỉ cần 200. Luôn thêm: Tối đa 200 từ. hoặc Chỉ 3 gạch đầu dòng.

3. Dùng input có cấu trúc, không dùng văn xuôi

Văn xuôi bắt AI phải phân tích ý nghĩa. Input có cấu trúc đưa thẳng vào:

  • Nhiệm vụ: review email
  • Bối cảnh: thanh toán trễ 30 ngày
  • Giọng điệu: cứng rắn nhưng không hung hăng
  • Output: chỉ email đã sửa, không giải thích

4. Đừng lặp lại context

Trong session dài, mọi người giải thích lại cùng một thứ ở mỗi tin nhắn. Nếu bạn đã nói 3 tin nhắn trước, nó vẫn còn trong context. Đừng nói lại.

5. Dùng caching cho system prompt lặp lại

Nếu dùng API: Claude, GPT-4, Gemini đều hỗ trợ prompt caching. System prompt 2.000 token được cache chỉ tốn ~10% giá bình thường mỗi lần gọi.

Tóm tắt:
Xóa lời chào → −20 token trung bình
Thêm giới hạn độ dài → −40% output token
Dùng input có cấu trúc → −25% prompt token
Ngừng lặp lại context → −30% trong session dài
Bật caching → −90% chi phí system prompt
